HT_info: Περιλαμβάνει των αριθμό των κάδων (buckets), το αναγνωριστηκό του φακέλου(fileDesc), τον μέγιστο αριθμό
εγγραφών που χωράνε σε ενα block(max_num_of_record_in_block),εναν πίνακα που έχει το τελευταίο block κάθε κάδου
(last_block_bucket_num[]), καθώς και αν είναι hash file η heap file (method == 2 hash file).

HT_block_info: Περιλαμβάνει των αριθμό των records(num_of_records) που έχει το συγκεκριμένο block καθώς και έναν δείκτη στο επόμενο
block του κάδου μέσω του index του block(next_bucket_block).


HT_CreateFile: Στην δημιουργία του φακέλου στο Block 0 περνάμε τα μεταδεδομένα (HT_info) με μία κατάλληλη αρχικοποίηση.
Στον πίνακα (last_block_bucket_num[]) σαν αρχικοποήση αναλόγος τα buckets που δίνονται σαν όρισμα σε κάθε θέση του πίνακα
βάζουμε τον αντίστοιχο αριθμό. Πχ για 10 κάδους ο παραπάνω πίνακας θα περιλμάνει τους αριθμόυς απο 1 εως 10 στις θέσεις από
0 εως 9(θα εξηγήσουμε αργότερα γιατί). Επίσης στην συνάρτηση αυτή εκτός από το block των μεταδεδομένων αρχικοποιούμε κατάλληλα
και το 1ο block του κάθε κάδου εισαγοντάς του και το αντίστοιχα ht_block_info. Έτσι γνωρίζουμε που βρίσκεται το 1ο block κάθε 
κάδου το οποίο θα μας διευκολύνει αρκετά στις επόμενες συναρτήσεις και μπορούμε να κάνουμε αρχικοποίηση του παραπάνω πίνακα.
(Απο άποψη waste of memory δεν υφίσταται γιατί αυτά τα αρχεία είναι για πολλά records οπότε σίγουρα κάθε κάδος θα περιλαμβάνει
ένα record.)

HT_OpenFile/HT_CloseFile: Αυτές οι συναρτήσεις είναι ίδιες με τις αντίστοιχες στο hp αρχείο.

HT_InsertEntry: Μέσω της hash function βρίσκουμε σε ποιόν κάδο πρέπει να μπεί το record. Μέσω του πίνακα που υπάρχει
στην ht_info βρίσκουμε ποιό είναι το τελευταίο block του κάθε κάδου. Αν το block δεν είναι γεμάτο κάνουμε την εισαγωγή
και αλλάζουμε μόνο το ht_block_info του συγκεκριμένου block. Αν είναι γεμάτο δημιουργούμε ένα καινούργιο block βάζουμε 
το record επίσης ενσωματόνουμε και το ht_block_infο διαμορφώνοντας το κατάλληλα. Επίσης αλλάζουμε το ht_block_infο του
προηγούμενου block του κάδου δηλαδή να δείχνει στο επόμενο block και στο ht_info αλλάζουμε τον πίνακα ώστε στο συγκεκρι-
μένη θέση του κάδου που έχουμε βρεί μέσω της hash function να υπάρχει το νέο τελευταίο Block. Επιστρέφουμε αναλόγος την
περίπτωση το block που έγινε η εισαγωγή.

HT_GetAllEntries:Μέσω της hash function βρίσκουμε σε ποιόν κάδο πρέπει να γίνει η αναζήτηση. Κάνουμε ενα loop που έχει
ως συνθήκη τερματισμού το block που ελέγχουμε να έχει ως next_bucket_block == -1. (Οταν κάνουμε αρχικοποίση σε ενα μποκ
είτε μέσω της create είτε μέσω της instert το next_bucket_block είναι -1 το οποία μας δείχνει οτι το συγκεκριμένο μπλοκ
είναι το τελευταία μπλοκ του κάδου). Με αυτόν τον τρόπο ψάχνουμε όλα τα block σειριακά του συγκεκριμένου κάδου. Επιστρέ-
φουμε έναν μετρητή με τα blocks του συγκεκριμένου κάδου που εξερευνήθηκαν.

Μέσα στην main έγιναν αναζητήσεις με διαφορετικά id επίσης είχαν προστεθεί και αρκετά print τα οποιά δείχναν την σωστή λειτουργία του κώδικα.
Οσο να αφορά τα Blocks ότι ισχυεί και στην hp_info.
